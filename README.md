# VMware 기반 가상 네트워크 보안 실습 환경 구축

## 1. 프로젝트 개요

VMware Workstation Pro를 이용하여 외부망과 내부 실습망을 분리하고,
Kali Linux, Windows, CentOS, Metasploitable2 등 여러 운영체제가 통신할 수 있는
가상 네트워크 보안 실습 환경을 구축하였다.

## 2. 프로젝트 목적

- 가상 네트워크 구조 이해
- Host-only와 NAT 네트워크의 차이 확인
- 정적 IP 주소 및 기본 게이트웨이 설정
- 라우팅 경로 확인
- 보안 실습을 위한 격리된 내부망 구축

## 3. 실습 환경

| 구분 | 운영체제 | IP 주소 | 역할 |
|---|---|---:|---|
| 공격 시스템 | Kali Linux | 추후 작성 | 패킷 분석 및 보안 실습 |
| 서버 1 | CentOS Stream 9 | 192.168.2.203 | Linux 서버 |
| 서버 2 | CentOS Stream 9 | 192.168.2.204 | Linux 서버 |
| 클라이언트 | Windows 7 | 192.168.20.202 | 내부망 클라이언트 |
| 서버 | Windows Server 2008 R2 | 192.168.20.201 | 내부망 서버 |
| 취약 시스템 | Metasploitable2 | 추후 작성 | 취약점 실습 대상 |

## 4. 네트워크 구성

- VMnet1 Host-only: 192.168.20.0/24
- VMnet8 NAT: 192.168.2.0/24
- 내부 게이트웨이: 192.168.20.100
- 외부측 인터페이스: 192.168.2.100
- VMware NAT Gateway: 192.168.2.254
- DNS 서버: 168.126.63.1

## 5. 네트워크 구조도

추후 네트워크 구조도 이미지 추가

## 6. 구축 과정

### 6.1 VMware 가상 네트워크 설정

VMnet1은 내부 실습망으로 사용하고,
VMnet8은 인터넷 연결을 위한 NAT 네트워크로 사용하였다.

### 6.2 운영체제별 IP 설정

각 가상 머신에 정적 IP 주소, 서브넷 마스크,
기본 게이트웨이 및 DNS 주소를 설정하였다.

### 6.3 통신 확인

다음 명령어를 사용하여 네트워크 연결 상태를 확인하였다.

```bash
ping
ip addr
ip route
arp -n
traceroute
nslookup
