# 네트워크 보안 실습 환경 구축

## 1. 실습 개요

네트워크 보안 실습을 진행하기 위해 VMware Workstation Pro 17을 이용하여 가상 네트워크 환경을 구축하였다.

외부 인터넷과 연결되는 NAT 네트워크와 내부 보안 실습에 사용하는 Host-only 네트워크를 분리하였으며, Windows, Linux, Kali Linux 등 여러 운영체제를 가상 머신으로 구성하였다.

본 실습 환경은 네트워크 통신 구조를 확인하고, 이후 패킷 분석 및 네트워크 보안 실습을 수행하기 위한 목적으로 구축하였다.

## 3. 가상 네트워크 구성

VMware의 Virtual Network Editor에서 다음과 같이 두 개의 가상 네트워크를 구성하였다.

| 네트워크 | 대역 | 용도 |
|---|---|---|
| VMnet1 | 192.168.20.0/24 | 내부 보안 실습망 |
| VMnet8 | 192.168.2.0/24 | NAT 기반 외부 연결망 |

VMnet1은 외부 인터넷과 직접 연결되지 않는 Host-only 방식으로 구성하였다.

VMnet8은 VMware의 NAT 기능을 이용하여 가상 머신이 호스트 컴퓨터의 인터넷 연결을 통해 외부 네트워크와 통신할 수 있도록 구성하였다.

---

## 4. IP 주소 구성

| 시스템 | IP 주소 | 역할 |
|---|---|---|
| Windows Server 2008 R2 | 192.168.20.201 | 내부 서버 |
| Windows 7 | 192.168.20.202 | 내부 클라이언트 |
| 내부 게이트웨이 | 192.168.20.100 | 내부망 트래픽 전달 |
| 외부측 인터페이스 | 192.168.2.100 | NAT망 연결 |
| CentOS Server 1 | 192.168.2.203 | Linux 서버 |
| CentOS Server 2 | 192.168.2.204 | Linux 서버 |
| VMware NAT Gateway | 192.168.2.254 | 외부 인터넷 연결 |
| DNS 서버 | 168.126.63.1 | 도메인 이름 조회 |

각 가상 머신에는 실습 목적에 맞게 정적 IP 주소, 서브넷 마스크, 기본 게이트웨이 및 DNS 서버를 설정하였다.

## 5. 네트워크 구조

내부망에 위치한 Windows 시스템은 `192.168.20.100`을 기본 게이트웨이로 사용한다.

내부망에서 발생한 트래픽은 게이트웨이를 거쳐 VMnet8의 NAT 네트워크로 전달되고, 이후 VMware NAT Gateway인 `192.168.2.254`를 통해 외부 인터넷으로 전송된다.

```text
Windows 7 / Windows Server
          │
          │ VMnet1
          │ 192.168.20.0/24
          │
  Gateway 192.168.20.100
          │
  External NIC 192.168.2.100
          │
          │ VMnet8
          │ 192.168.2.0/24
          │
 VMware NAT Gateway
     192.168.2.254
          │
       Internet

## 6. 네트워크 설정 확인

가상 머신의 네트워크 설정이 정상적으로 적용되었는지 확인하기 위해 다음 명령어를 사용하였다.

### Linux

```bash
ip addr
ip route
route -n
arp -n
ping
traceroute
nslookup
```

### Windows

```cmd
ipconfig
route print
arp -a
ping
tracert
nslookup
```

`ping` 명령어를 이용하여 가상 머신 간 통신 여부를 확인하였다.

`traceroute`와 `tracert`를 이용하여 패킷이 목적지까지 이동하는 경로를 확인하였으며, `nslookup`을 이용하여 DNS 서버가 도메인 이름을 IP 주소로 변환하는 과정도 확인하였다.

---

## 7. 통신 경로 확인

Kali Linux에서 외부 DNS 서버인 `168.126.63.1`을 대상으로 `traceroute`를 수행하였다.

```bash
traceroute 168.126.63.1
```

확인된 주요 경로는 다음과 같았다.

```text
192.168.20.100
→ 192.168.2.254
→ 외부 네트워크
```

첫 번째 홉인 `192.168.20.100`은 내부망의 기본 게이트웨이이다.

두 번째 홉인 `192.168.2.254`는 VMware의 NAT Gateway이다.

이를 통해 내부망에서 생성된 패킷이 내부 게이트웨이를 거친 뒤 VMware NAT 네트워크를 통해 외부로 전달되는 것을 확인하였다.

## 8. 실습 중 발생한 문제

### 8.1 Host-only 환경에서 외부 통신이 가능했던 현상

Host-only 네트워크는 기본적으로 호스트 컴퓨터와 가상 머신 사이의 내부 통신을 위한 네트워크이다.

하지만 게이트웨이 역할을 하는 가상 머신에 VMnet1과 VMnet8 네트워크 어댑터가 함께 연결되어 있고, 라우팅 기능이 설정되어 있으면 VMnet1의 내부 시스템도 외부 네트워크와 통신할 수 있다.

이를 확인하기 위해 라우팅 테이블과 `traceroute` 결과를 비교하였다.

```bash
ip route
route -n
traceroute 168.126.63.1
```

그 결과 내부망의 트래픽이 `192.168.20.100`을 거친 뒤 VMware NAT Gateway인 `192.168.2.254`로 전달되는 것을 확인하였다.

### 8.2 구형 Windows에서 VMware Tools 설치 실패

Windows 7과 Windows Server 2008 R2에서 VMware Tools를 설치하는 과정에서 일부 드라이버가 정상적으로 설치되지 않는 문제가 발생하였다.

구형 Windows가 최신 드라이버의 SHA-2 코드 서명을 지원하지 않는 것이 원인일 가능성이 있어 다음 업데이트를 설치하였다.

```text
KB4490628
KB4474419
KB3140245
```

업데이트 설치와 재부팅을 진행한 후 VMware Tools 설치를 다시 수행하였다.

이를 통해 클립보드 공유와 화면 크기 조절 등 가상 머신 편의 기능을 사용할 수 있도록 구성하였다.

- ARP Cache 변화 확인
- 내부망과 외부망의 패킷 흐름 비교
- 네트워크 공격 및 방어 트래픽 분석
